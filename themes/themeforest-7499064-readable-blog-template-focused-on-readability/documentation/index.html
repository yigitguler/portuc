Scaffolding

Build on top of Twitter Bootstrap 3.0

The Readable HTML template is build on top of SASS Twitter Boostrap HTML framework, using modern tools like Grunt, Bower and npm. All the components of the Bootstrap framework were left as they are during the development, so the template is upgrade-safe.

Find the documentation for Twitter Boostrap HERE.

You are allowed to use any component in your themes available in Twitter Bootstrap. However, for the performance reasons some of the components were commented out in the src/sass/main.scss file. Uncomment the components you need in your project and recompile the project using $ grunt or $ grunt build when you cd to the root of the template in terminal.

Tools used

  • Grunt with the following tasks:
    • matchdep - just loads the dependencies listed in the package.json file.
    • watch + liverealod - watch the file, run the task when the files change and trigger the autoreload in the browser.
    • compass - used for compiling the SCSS/SASS files to CSS.
    • autoprefixer - the up-to-date smart prefixing utility. No need to write the prefixed CSS3 properties.
    • assemble - solves the nightmare of the static HTML files. Simple templating engine using HandlebarsJS. With partials, data, helpers etc.
    • imagemin - losselessly optimizes the images for the production.
    • newer - speeds up the building process by running tasks only on the files that actually changed.
    • concurrent - used when multiple blocking watch tasks run at the same time. For watch and compass.
    • usemin with:
      • concat - concatenates multiple JS/CSS files into a single file.
      • cssmin - minifies the CSS files.
      • rev - prepends the hash to the files for caching bust.
    • requirejs with almond for build/ and minified using UglifyJS2 - loads the AMD modules, optimizes them (concatenating, minifying), concatenates and saves the output.
    • clean - cleans the directories, we love to have everything tidy.
  • Handlebars.js - JavaScript templating engine, used together with assemble Grunt task
  • jQuery - JS library
  • Bootstrap JS - JS functions for Twitter Bootstrap
  • Underscore.js - Library of JS helper functions

Using Grunt (optional)

Readable is build with the advanced tools for the frontend development. Grunt is a JavaScript task runner. The Gruntfile.js comes with the theme, so one can optimize all the files and code with little to no effort.

If you want to edit the HTML files directly, you can edit the build sources in the build directory or more loosy HTML files in the src directory. However, if you are familiar with the Grunt and Bower, you can just run the following commands in the terminal to start using the prepared Gruntfile:

$ cd <the-directory-of-readable>
$ npm install
$ bower install

That should install all the dependencies in the folders node_modules and src/bower_components. For the development work you can then just run:

$ grunt

It has live reload as well, just enable it in the Chrome Extensions. For building the sources to the highly-optimized code, run in terminal:

$ grunt build

That will export all the sources to the folder build which is then ready for production.

  • Open Sans - for the headings
  • Lato - for meta and smaller text
